# Preferences

## Coding Style

Clean, simple, human-readable code that is explicit.

- Prefer explicit over implicit
- Prefer simple over clever
- Prefer readable over compact
- Strong type guarantees
- Generated API documentation

## API Design

Prefers REST over GraphQL. The simpler and more explicit, the better.

- REST over GraphQL
- Explicit contracts and schemas
- Generated documentation from types/schemas
- Clear, predictable endpoints

## Tooling

Runtime of choice is TypeScript / Node.js.

- TypeScript for type safety and strong guarantees
- Minimal abstractions — only add complexity when earned
- Explicit configuration over convention-based magic
